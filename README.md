# bigdataprogramming
2026 myongji university project

1. 문제 정의
   
- 해결하고자 하는 문제: 공공 진료 데이터를 기반으로 계절/월별 특정 질환(ex: 호흡기 질환, 온열 질환)의 발병 추이를 파악합니다. 또한 지역별 의료 인프라(병원 수)와 실제 진료 이용량 간의 불균형이 존재하는지 확인합니다. 최종적으로 연령대별 다빈도 질환의 특징과 진료비 지출 규모의 상관관계를 분석하여 의미 있는 인사이트를 도출하고자 합니다.
- 사용 데이터: 공공데이터포털(data.go.kr)에서 제공하는 건강보험심사평가원 오픈 API를 활용하겠습니다. 100MB 이상의 데이터를 확보하여 신뢰성이 높은 인사이트를 도출해내겠습니다.


2. 기술 스택

- 데이터 수집: Python(오픈 API 호출 및 수집 자동화 스크립트 작성)
- 데이터 저장: HDFS(수집된 원본 데이터 및 전처리된 데이터 저장)
- 데이터 전처리 및 분석: Spark SQL, Hive(대용량 데이터의 결측치 처리, 조인, 집계 등 배치 처리 수행)
- 시각화: Matplotlib, Seaborn 등 (분석 결과 그래프화 및 대시보드 구성)


3. 구현 계획(데이터 파이프라인)

- 1단계(데이터 수집): Python 스크립트를 작성하여 공공데이터포털 API를 주기적으로 호출하여 데이터를 수집합니다. 이 과정은 재실행 가능하도록 스크립트화합니다.
- 2단계(데이터 저장 및 전처리): 수집된 원본 데이터를 CSV등의 포맷으로 HDFS에 저장합니다. 이후 Spark DataFrame이나 Hive Query를 활용하여 결측치 처리, 타입 변환 등 분석에 알맞은 형태로 전처리합니다.
- 3단계(데이터 분석): 전처리된 데이터를 바탕으로 Spark SQL과 HiveQL을 활용합니다. 단순 데이터 조회가 아닌 GROUP BY, JOIN 등을 적극 활용하여 앞서 정의한 3가지 핵심 질문에 대한 답을 분석합니다.
- 4단계(결과 시작화): 도출된 분석 결과를 이해하기 쉽도록 시각화 도구를 활용해 그래프와 표로 표현하고 최종결과를 정리합니다.
